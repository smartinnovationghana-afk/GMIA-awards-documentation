### 1. Header & User Welcome Section

At the top of the dashboard, the system displays a personalized welcome message: **"Welcome Marcell"**. Adjacent to the user's name are two identifier badges — a role indicator labeled **"manager"** and an organizational affiliation badge displaying **"Asanko Gold Ghana Ltd."**. To the far right of the header, a notification bell icon and a user avatar (marked with the initial "M") provide quick access to alerts and profile settings.

***

### 2. Plans & Quick Actions Section

Beneath the welcome header, a section titled **"What are your plans today?"** serves as the primary engagement area. A brief subtitle explains: _"This platform is designed to for users from the chambers company"_ (sic).

This section presents a horizontal row of four quick-action cards, each enabling rapid navigation to core data upload functions:

* **GCM Tech Stats** — Represented by a gear-and-chart icon, this card allows users to upload technical statistics for the Ghana Chamber of Mines.
* **GCM HR Stats** — Identified by a human-resources icon, this card facilitates the upload of human resource statistics.
* **Security Report** — Marked by a shield-and-warning icon, this card enables the submission of monthly security reports.
* A fourth card (partially visible) appears to provide additional upload functionality.

***

### 3. Notifications Panel

Positioned in the left-center area of the dashboard, the **Notifications** panel provides a centralized hub for system alerts. The panel header includes three action controls: an **"All"** filter toggle, a **"Mark All"** option to dismiss all notifications, and a **"Clear"** button to remove notifications from view.

When no notifications are present, the panel displays the message: _"No notifications found."_

***

### 4. Latest Uploads Panel

Adjacent to the Notifications panel, the **Latest Uploads** section tracks recent data submissions across the platform. A **"Manage All"** link in the header provides access to the full upload history.

The panel displays upload records by module, with the following current status:

* **Technical Stats** — Shows a submission titled _"First Quarter upload from a particular company."_ dated **09 Jun, 2026**, with a **"PENDING"** status badge. A **"View"** link allows detailed inspection of the submission.
* **HR Stats** — Displays _"No upload from this model yet"_.
* **Security Report** — Similarly indicates _"No upload from this model yet"_ .

***

### 5. Quarterly Uploads Panel

Located on the right side of the upper dashboard, the **Quarterly Uploads** panel provides a quarter-by-quarter overview of organizational compliance and data submission status. The panel header displays **"2026 — last upload per module"** with quarter-selector tabs for **Q1, Q2, Q3, Q4**, and an **"All"** aggregate view.

Each module is represented as a distinct card containing:

* A module icon (e.g., document icon for Technical Stats)
* The module name
* The most recent upload description
* The upload date and submitting organization
* A status indicator (e.g., **"PENDING"**, **"NONE, APPROVED"**)
* A **"View"** navigation link

Current module statuses include:

* **Technical Stats** — _"First Quarter upload from a particular company"_ dated **09 Jun 2026**, status: **PENDING**
* **HR Stats** — _"No upload this quarter"_, status: **NONE**
* **Security Report** — _"No upload this quarter"_, status: **NONE**
* **Incident Report** — _"No upload this quarter"_, status: **NONE**
* **Dangerous Occurrence** — Partially visible, indicating additional compliance tracking modules

***

### 6. Calendar & Meetings Section

In the lower-left portion of the dashboard, a **June 2026** calendar widget provides a monthly view of scheduled activities. Navigation arrows allow month-to-month traversal. The current date (**29**) is highlighted in the calendar grid.

Beneath the calendar, a status message reads: _"No meetings scheduled for this month."_ This indicates that the calendar integrates with the platform's meeting and event management functionality.

### 9. Floating Action Button

A prominent circular floating action button (blue, with an upload/download icon) is positioned at the bottom-right corner of the dashboard interface. This persistent element provides one-click access to primary upload or download functionality regardless of scroll position.
